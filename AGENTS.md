# AGENTS.md

## 项目概况
- MCU: STM32F103C8T6
- 工程由 STM32CubeMX 生成，当前源码位于 `Core/`、`Com/`、`Drivers/`
- 当前主要调试口为 `USART1`，引脚为 `PA9/PA10`

## 当前已确认事实
- 板子硬件基本已通过交叉验证排除，串口不输出更可能是工程初始化或重定向问题
- `SystemClock_Config()` 当前依赖 `HSE` 外部时钟
- 初始化顺序里，若时钟或 RTC 失败，会在串口常规 `printf` 之前进入 `Error_Handler()`

## 代理工作约定
- 修改 CubeMX 生成文件时，优先放在 `USER CODE` 区域，避免后续重新生成时丢失
- 在未确认问题根因前，优先做最小侵入式定位，不做大规模重构
- 串口调试优先区分三类问题：
  1. 程序未跑到 UART 初始化
  2. 程序跑到了 UART，但 `printf` 重定向未生效
  3. 程序已发送，但外部观察链路有问题
- 若新增定位代码，完成定位后应提醒用户清理临时调试输出
